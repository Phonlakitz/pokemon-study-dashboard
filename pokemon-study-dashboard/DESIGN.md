# Pokémon Study Dashboard
Design System & Product Guidelines

---

# Product Identity

This project is NOT a todo app.

This project is a cozy study operating system inspired by:

- Pokémon Nintendo DS menus
- Pokédex interfaces
- Nintendo handheld consoles
- retro RPG menu systems
- cozy productivity tools

The user should feel like:

"I am managing my trainer journey through studying."

Studying = Quests

Progress = EXP

Consistency = Leveling Up

---

# Core Product Goals

The application should feel:

- Cozy
- Motivating
- Rewarding
- Nostalgic
- Organized
- Playful

Never feel:

- Corporate
- SaaS-like
- Minimalistic startup UI
- Enterprise dashboard

---

Rules

- Titles
- Navigation
- Labels
- Status tags
- Buttons

---

# Border System

The entire application uses a layered border style.

Panels should feel:

- chunky
- tactile
- physical

Every major container should have:

- outer border
- inner border
- inset highlight

Inspired by Nintendo DS UI frames.

---

# Corner Radius

Use:

12px - 20px

Avoid:

sharp corners

glassmorphism

modern rounded-pill styles

---

# Navigation Design

Bottom navigation is one of the most important parts of the app.

It should feel like:

a Nintendo DS game menu.

Requirements:

- chunky tabs
- layered borders
- compact spacing
- pixel labels
- centered icons

Selected tab:

- red accent border
- light cream background
- stronger contrast

Inactive tabs:

- soft blue-gray

Never redesign navigation into a modern web navbar.

---

# Icon Design

Icons should be:

- pixel art
- retro
- clean
- readable

Use:

32x32

48x48

64x64

Do not mix styles.

All icons must belong to the same pixel-art family.

---

# Dashboard Philosophy

Dashboard = Mission Control

The user should immediately see:

- Progress
- Weekly Quests
- Current Status

Hierarchy:

1. Weekly Progress
2. Daily Quest Cards
3. Navigation

---

# Daily Quest Cards

Quest cards should resemble:

Pokémon menu panels.

Include:

- Day icon
- Day label
- Quest list
- Quest count
- Status

Use:

soft pastel colors

large spacing

clean hierarchy

subtle Poké Ball watermark

---

# Add Quest Screen

Add Quest screen should feel like:

Preparing missions before an adventure.

Layout:

Input Panel

RIGHT
Quest Queue

# Form Controls

Nintendo DS

# EXP System

EXP is a core emotional mechanic.

EXP bars should:

- animate smoothly
- feel rewarding
- look RPG-inspired

Avoid:

plain HTML progress bars

---

# Animation Philosophy

Subtle only.

Duration:

150ms - 250ms

Allowed:

hover

button press

quest completion

progress fill

tab switching

Avoid:

large motion

modern floating effects

excessive bouncing

---

# Emotional UX

The user should feel:

"I completed something today."

Every completed quest should provide:

small satisfaction

small dopamine

small sense of progress

The app should encourage consistency through comfort.

Never through pressure.

---

# Future Features

Future additions must match the Pokémon Study OS identity.

Allowed:

EXP

Levels

Streaks

Badges

Trainer Rank

Companion System

Focus Timer

Weekly Reset

Achievements

Quest History

Pokédex-style Statistics

---

# Golden Rule

Whenever designing a new screen:

Ask:

"Would this feel natural inside a Nintendo DS Pokémon game menu?"

If the answer is no,

redesign it.
