# Pokémon Inspired Study Dashboard — Design System

---

# Design Philosophy

This project is NOT a normal productivity dashboard.

The goal is to create:
"A cozy Nintendo DS inspired study operating system."

The experience should feel:

* nostalgic
* emotional
* soft
* game-like
* satisfying
* comforting

The user should feel like they are:

* managing weekly quests
* preparing for an adventure
* leveling up through studying

This project prioritizes:

* atmosphere
* emotional design
* cozy interactions
* visual identity

Over:

* enterprise UX
* feature overload
* modern SaaS aesthetics

---

# Main Inspirations

Inspired by:

* Pokémon Nintendo DS menus
* Pokédex interfaces
* Nintendo DS UI
* retro handheld consoles
* cozy pixel-art games

DO NOT:

* directly copy Pokémon UI
* use copyrighted assets
* mimic exact layouts

Instead:
Create an original interface with similar emotional energy.

---

# Core Visual Direction

The UI should feel:

* chunky
* tactile
* layered
* playful
* soft
* handheld-console-like

The interface should feel:

* touchable
* button-based
* cozy
* retro

---

# Color Philosophy

Colors should:

* feel pastel
* avoid harsh contrast
* remain soft on the eyes
* create emotional warmth

Avoid:

* neon colors
* pure black
* oversaturation
* overly modern gradients

# Typography Rules

## Pixel Fonts

Use ONLY for:

* titles
* section labels
* navigation
* badges

Recommended:

* Press Start 2P
* Pixelify Sans
* VT323

DO NOT:

* use pixel fonts for paragraphs
* use tiny unreadable text

---

# Body Text

Use cleaner readable fonts for:

* quest descriptions
* task text
* paragraphs
* UI details

The app should remain comfortable for long study sessions.

---

# Panel Design Rules

Panels should:

* feel layered
* feel tactile
* resemble Nintendo DS menus

Use:

* rounded-2xl corners
* thick borders
* inset shadows
* layered backgrounds
* subtle gradients

Avoid:

* flat modern cards
* glassmorphism
* ultra-minimalism

---

# Card Design Rules

Quest cards should:

* feel collectible
* have personality
* feel slightly raised

Use:

* soft shadows
* layered borders
* hover lift
* subtle glow

Hover behavior:

* translateY(-2px)
* soft border glow
* slight brightness increase

---

# Button Philosophy

Buttons should feel:

* chunky
* physical
* tactile
* pressable

Buttons should resemble:

* Nintendo DS menu buttons
* retro handheld interfaces

Use:

* inset shadows
* layered borders
* slight press animation

Press effect:

* scale(0.98)
* darker shadow
* lower brightness

---

# Navigation Design

Bottom navigation should feel:

* game menu-like
* handheld console inspired
* emotionally cozy

Include:

* rounded menu tabs
* selected glow
* red active state
* Poké Ball inspired central action button

Navigation should NOT:

* resemble mobile banking apps
* feel modern corporate
* use sharp geometry

---

# Animation Philosophy

Animations should:

* be subtle
* feel game-like
* remain lightweight

Use:

* 150ms–250ms transitions
* ease-out timing
* smooth hover interactions

Avoid:

* excessive motion
* flashy modern effects
* aggressive bounce animations

---

# Task Interaction Rules

When a task is completed:

* checkbox animates
* text receives line-through
* opacity lowers slightly
* completion feels rewarding

The interaction should create:
small dopamine feedback

---

# EXP System Visual Direction

EXP bars should:

* feel rewarding
* resemble RPG progression

Use:

* smooth animated fills
* subtle gradients
* glossy overlays
* soft highlights

Avoid:

* flat web-style progress bars

---

# Background Philosophy

Backgrounds should:

* support the UI
* never overpower content

Use:

* subtle texture
* low-contrast patterns
* muted gradients

Avoid:

* noisy patterns
* overly bright backgrounds
* strong contrast textures

---

# UI Density

The dashboard should:

* breathe comfortably
* avoid overcrowding

Spacing should feel:

* relaxed
* clean
* cozy

Use:

* generous padding
* balanced spacing
* grouped UI sections

---

# Emotional UX Goals

The user should feel:

* calm
* motivated
* organized
* emotionally rewarded

The app should encourage:

* returning daily
* enjoying study sessions
* building consistency

NOT through pressure,
but through comfort and atmosphere.

---

# Development Philosophy

Build gradually.

DO NOT:

* overengineer
* rebuild repeatedly
* chase perfection immediately

Instead:

* improve systems first
* polish interactions second
* refine visuals gradually

Preserve:

* atmosphere
* identity
* emotional consistency

at every stage.

---

# Long-Term Identity

This project should eventually feel like:

"A cozy handheld RPG where studying is the adventure."
